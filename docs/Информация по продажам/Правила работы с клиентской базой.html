Регламент работы с клиентской базой

Общие положения

Закрепление клиентов осуществляется в Битрикс. В 1С при первой продажи карточка клиента заводится карточка контрагента.

Клиент закрепляется за одним менеджером в двух системах. В каждой карточки Компании должна стоять актуальная (не просроченная) задача. Если менеджер не смог перевести клиента в сделку, либо по какой-то причине менеджер не ведет клиента дальше, то клиент перезакрепляется на аккаунт Свободный клиент.

Действующий клиент – клиент, закрепленный за менеджером, который осуществлял покупку в течение последних 3 месяцев, в карточке Компании по нему стоит актуальная задача.

Потенциальный клиент – клиент из ЦА, которого можно перевести в сделку, новый клиент, не закрепленный за менеджером, либо компания, не покупающая последние 3 месяца и без задачи ответственного менеджера.

Свободный клиент – потенциальный клиент, не закрепленный ни за одним менеджером.

Свободный Потенциальный и Архивный клиент

В Битрикс есть 3 дополнительных аккаунта, Свободный, Потенциальный и Архивный клиент.

На каждый аккаунт закрепляется база, согласно регламенту

Свободный клиент – аккаунт, за которым закрепляются клиенты, в случае, если менеджеру не удалось выйти на ЛПР, перевести клиента в сделку в течение месяца, либо по своей иной причине менеджер не готов дальше вести клиента

Потенциальный клиент – аккаунт, за которым закрепляются новые Лиды

Архивный клиент – аккаунт, за которым закрепляются компании, по которым больше не будет никаких продаж – пример: компания закрылась, ликвидировалась.

Правила распределения клиентов

  • Менеджер ведет работу с закрепленными за ним с компаниями или с потенциальными компаниями. В карточке каждой компании, закрепленной за менеджером, должна стоять актуальная задача.
  • Клиент, который покупал у компании продукт не ранее 3 месяцев, является действующим клиентом. С действующими клиентами ведет работу ответственный менеджер.
  • Счет потенциальному клиенту может выставлен любым менеджером
  • Если с клиентом не было отгрузок более 3 месяцев и в карточке Компании нет актуальной задачи, клиент становится Потенциальным.
  • Дочерняя/дополнительная организация, подразделение организации с другим КПП всегда закрепляется за тем же менеджером, который ведет работу по головной организации.
  • Если менеджер начал работать по карточке компании с нарушением правил распределения клиентов, то дальнейшая сделка принадлежит ответственному менеджеру

Клиентская база менеджера формируется несколькими путями:

  • руководитель отдела продаж предоставляет менеджеру готовую клиентскую базу для проработки клиентов по определенной задаче, например, обзвон отказников;
  • менеджер самостоятельно ищет компании в интернет и пополняет клиентскую базу, соблюдая регламент;
  • при увольнении сотрудника руководитель отдела продаж распределяет базу по другим менеджерам, до распределения все клиенты считаются потенциальными.

Проверка закрепления карточки контрагента

Проверка закрепления и состояния клиента с точки зрения отношений с ним

Прежде чем начать работу с клиентом, менеджер должен убедиться, что клиент является свободным. Любой менеджер может начать работу с потенциальным клиентом.

Взять клиента в работу (в том числе и потенциального) можно, только если с ним не ведет работу другой менеджер по головной или дочерней компании.
